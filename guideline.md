# 具身智能引论期末课程项目介绍

## 我们准备怎么实现这个愿景？
课程项目的核心，是搭建一个 multi-agent 系统：语音交互 agent 作为与用户交互的入口，视觉 agent 负责理解、检测与定位，机械控制 agent 负责生成并执行轮子与机械臂的动作。

### 语音交互 Agent
作为用户的主要交互接口，负责完成唤醒、语音识别、对话生成、功能调用、语音合成与播放。它既要听懂用户在说什么，也要决定系统下一步该调用哪些能力。

### 视觉 Agent
提供视觉理解、检测与定位能力。例如理解场景里有哪些物体、识别用户行为、检测人脸位置，以及结合机械臂运动估计视觉目标在三维空间中的位置。

### 机械控制 Agent
根据任务目标与视觉反馈，生成并执行轮子和机械臂的控制指令。例如控制移动底盘改变位置、控制机械臂到达某个姿态，或者将末端夹爪移动到某个目标点。

### System workflow
用户通过语音和语音交互 Agent 沟通。
视觉 Agent 为语音交互 Agent 提供环境上下文，例如当前看到了什么、人在哪里、目标物体在哪里。
语音交互 Agent 可以发起功能调用，例如追踪、触碰、递物等等。每个功能调用都对应一个视觉理解 + 机械控制的小闭环。

### Example 1 · 追踪任务的闭环
›
语音交互 agent 接收到用户关于“跟随我”或“看着我”的请求。
›
视觉 agent 调用人脸或物体检测算法，返回目标检测框或目标位置。
›
机械控制 agent 根据目标在图像中的位置，调整轮子运动方向或机械臂末端位姿。
›
新的动作带来新的视觉观测，视觉 agent 继续更新目标位置，系统形成实时追踪闭环。

### Example 2 · 触碰任务的闭环
›
语音交互 agent 接收到“去碰一下某个目标”的指令，并将触碰目标传递给视觉 agent。
›
视觉 agent 先通过场景理解与检测模型找到目标，再调用机械臂去左右移动、主动收集与该目标相关的数据。
›
视觉 agent 结合观测与运动信息，估计目标点在三维空间中的位置。
›
机械控制 agent 根据三维位置规划动作并驱动机械臂执行触碰；执行过程中还可以通过视觉反馈进一步修正。

## 课程项目安排
整个课程项目分为两个阶段。第一阶段统一要求大家完成基础功能，第二阶段鼓励大家在此之上自由发挥。

### 第一阶段：完成基础 multi-agent workflow
时间范围：第 8 周到第 12 周。 各小组需要按照课程给出的 agent 框架，完成三个 agent 与完整的 workflow。

Workflow 至少需要支持四个基础功能：语音交互、视觉场景识别、人脸或物体追踪、物体触碰。

换句话说，第一阶段的重点不是只完成某个局部模块，而是搭建一个真正能跑起来的系统闭环。

### 第二阶段：自由扩展与重构
时间范围：第 12 周到第 16 周。 完成基础系统后，各小组可以自由引入自己感兴趣的新功能。

你们也可以修改整个 multi-agent 框架，不必被课程提供的基础框架限制住。

只要是在基础功能之上继续开发，具体方向不做限制，欢迎大家自由发挥，做出各组独特的系统设计。

## 第一阶段详细要求与评估标准

第一阶段的重点，是让 multi-agent 系统真正支持用户通过自然语音发起请求，并完成从理解、功能调用到执行反馈的完整闭环。评估不仅看功能能否跑通，也看交互是否自然、稳定、流畅。

### 1. 语音交互
要求 Agent 能够和用户通过语音进行对话、聊天。系统应具备从语音输入到语音回复的完整流程，包括语音识别、回复生成以及语音合成。大家也可以直接使用端到端的语音模型来实现。

### 2. 功能调用
当用户通过语音发出调用功能的请求，例如“碰一下那个开关”之后，Agent 应能够自动调用相应的功能。

#### (1) 视觉追踪
发出跟随物体的语音指令，例如“看我的电脑屏幕”，机械臂能够转向该物体。随着这个物体的移动，机械臂也会即时跟随它移动，保持该物体在腕部相机视野的中央区域。物体类型不限。

#### (2) 视觉识别
发出语音指令，例如“我正在电脑上做什么”，机械臂能理解画面中的内容，并生成对话回复。

#### (3) 物体交互
发出语音指令，例如“碰一下那个开关”，机械臂能够完成对开关位置的定位，并控制各关节的移动，使末端夹爪能够碰到它。

### 评价指标

#### (1) 功能性
包括：(1) Agent 生成语音的自然程度；(2) 能否支持模糊的语音指令，而非硬编码的字符匹配；(3) 能否稳定完成对物体的追踪或触碰。

#### (2) 流畅度
要求从语音对话或指令发出，到 Agent 产生语音回复或产生执行指令的动作之间的时间越短越好。执行指令过程中，感知到的卡顿越少越好。
